Week 7

Date: 04/05/2026

Project Summary:

The project I'm working on is intended to be a mock of a private enterprise network running on Cisco's Packet Tracer. This project was chosen as it would be a good stepping stone into Network Engineering as well as it would've allowed me to have a better grasp on how the OSI/TCP model works to make the internet what it is today. It's also worth noting that when I planned to build this project out, I intended to also allocate a little less than half of my project time to participating in a CCNA prep course which would've allowed me to get a better grasp of some of the features of my network I intended to implement while simultaneously preparing for the CCNA exam.

Accomplishments and Goals:

The features I intended to include in my network, as seen below, include DHCP, NAT/PAT configuration, DNS, and TACACS+ Authentication. So far I've made progress as I've finalized my network design, built a basic private network with multiple subnets, as well as pretty much completed implementing DHCP in my enterprise network (ability to automatically assign IP addresses). On the CCNA preparation front, I was able to make significant progress deep diving into topics like LAN switching, IPv4 addressing, working with switch interfaces, and other less significant topics, but as I will explain below in my changes section, the time I've been spending following a CCNA course will need to be allocated to building out the network. As stated in my proposal, my goal is to have a working network in Cisco Packet Tracer with the features listed below so that I'm able to test and run analysis to be discussed in a demonstration.

Private Enterprise Network Progress and Goals:

  • Foundation and Network Design
  • Build a basic Network
  • Implement DHCP
  • Create NAT/PAT configuration
  • Research and Implement DNS & Advanced Services
  • Implement Wireless Infrastructure
  • Research and Implement TACACS+ Authentication
  • Run testing, verification, security hardening
  • VLAN Implementation
  • VLAN Testing & Validation
  • Clean up documentation, Demo Video, and Report

Changes:

For the first two weeks of the project I was able to follow Jeremy's IT Lab's CCNA course as well as design a network topology and implement DHCP in my private network. After those weeks, I realized quickly that my workload was increasing as my other courses became more demanding and I began interviewing with companies more sporadically. Instead of supplementing my network development with a strict following of a structured course, I've now decided to focus solely on building my network and researching features as needed for my project. I still plan on using the CCNA course for research when needed as the material is very digestible and well laid out. That being said, as I've fallen significantly behind on building out other network functionality, specifically setting up NAT/PAT configurations, I may have to push out implementing VLANs in my network as I initially intended to do as a mitigation strategy if I started falling behind.

Reflection:

Overall, I do feel like I can comfortably implement all the features in my private network in Cisco Packet Tracer, conduct the necessary testing and analysis on my network, and demonstrate what I've built by April 24th. Reflecting on my initial few weeks, I'm very confident that spending less time on a CCNA course and pushing off implementing VLANs in my project will make accomplishing the rest of my goals more realistic by the deadline. While I like the CCNA course I've taken, I wish I hadn't pushed myself to follow it sequentially with 1-2 hours of my time each week and instead used it when necessary for my project. I do feel I could've at least made significantly more progress implementing NAT/PAT configurations in my network by now.
